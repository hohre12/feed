# English Writing Rules

Applied when `language=en`. These rules shape how English-language SNS content should sound.

---

## Base Tone

- Casual, conversational, lowercase OK
- Talking to yourself, not presenting to an audience
- Stream of consciousness — like texting a thought to your notes app at 2am
- Not "writing content." Just... thinking out loud

---

## Character Limits

- One line max **~40 characters** (optimized for mobile vertical scroll)
- Total **3-7 lines** (excluding blank lines)
- For long-form platforms (Reddit, Dev.to), these limits relax — follow platform spec instead

---

## Required Colloquial Patterns

### Contractions (mandatory)

Never use full forms in casual content. Always contract.

| Do | Don't |
|------|------|
| don't | do not |
| can't | cannot |
| it's | it is |
| won't | will not |
| would've | would have |
| should've | should have |

### Informal Shortenings (0-2 per post, don't overdo it)

Use sparingly for natural flavor:

- gonna, wanna, kinda, sorta, gotta, tbh, ngl, idk, imo
- "lowkey", "highkey", "literally", "basically"

| Good | Bad |
|------|------|
| "gonna pretend I didn't see that" | "I am going to pretend I did not see that" |
| "kinda wish I never learned this" | "I somewhat wish I had never learned this" |

### Incomplete Sentences

Drop subjects, trail off, leave thoughts hanging.

| Good | Bad |
|------|------|
| "just... why." | "I simply do not understand why." |
| "not this again." | "This is happening again." |
| "oh." | "Oh, I see." |

### Trailing Off

Sentences that don't quite finish — thought still in progress.

- "...but like, why though"
- "...and then I just"
- "...idk man"
- "...right?"

### Casual Sentence Endings

How real people end thoughts online:

- "honestly" (as a standalone or sentence ender)
- "or whatever"
- "but yeah"
- "so there's that"
- "anyway"

---

## Forbidden Patterns

These instantly break the casual tone. Never use them.

| Type | Forbidden Example | Why |
|------|-------------------|------|
| Formal openers | "In this post, I will discuss..." | Academic / corporate |
| Essay transitions | "Furthermore", "Moreover", "Additionally" | Sounds like a term paper |
| Passive voice (overuse) | "It has been observed that..." | Nobody talks like this |
| Perfect grammar | Every sentence perfectly punctuated and complete | Too polished = AI smell |
| Motivational cliches | "hustle harder", "rise and grind" | LinkedIn energy, instant cringe |
| Corporate speak | "leverage", "synergy", "at the end of the day" | Reads like a press release |
| Over-hedging | "I think perhaps it might be the case that..." | Just say the thing |
| Numbered lists in short-form | "1. First, 2. Second, 3. Third" | Save structure for long-form |

## Good vs Bad Examples

### Good Example

```
spent 3 hours debugging
it was a typo
it's always a typo
```

> Incomplete sentences. Lowercase. Repetition for humor effect.
> Feels like a real dev venting, not a content piece.

### Good Example

```
normalize quitting things
that aren't working
instead of romanticizing
the struggle
```

> Line breaks create rhythm. No capitalization. Casual imperative.
> Reads like a genuine thought, not a motivational poster.

### Bad Example

```
I spent three hours debugging my code today.
It turned out to be a simple typo.
This experience reminded me of the importance of attention to detail.
```

> Every sentence is grammatically complete. The "lesson learned" ending
> is classic AI/LinkedIn energy. Nobody posts like this unironically.

### Bad Example

```
In today's fast-paced world,
we often forget to take a step back
and appreciate the little things.
```

> Generic. Could be about anything. "In today's fast-paced world" is
> the English equivalent of writing "dear diary." Instant scroll-past.
