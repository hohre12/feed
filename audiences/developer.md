# 개발자 (Developer)

## 검색 소스 (가장 중요)
### 주제 없을 때 (소재 발굴)
- 1순위: GitHub Trending (오늘/이번 주 트렌딩 저장소)
- 2순위: Hacker News Top/Best
- 3순위: Dev.to Trending
- 4순위: Reddit r/programming, r/webdev

### 주제 있을 때 (팩트 검증)
- 1순위: GitHub (저장소, star 수, 최신 릴리스)
- 2순위: 공식 문서 (해당 기술의 docs)
- 3순위: Stack Overflow
- 4순위: 기술 블로그 (공식 블로그, engineering blog)

## 톤
- 기술 용어 그대로 사용 (CI/CD, ORM, 번들러 등 번역 X)
- "이거 써봤는데" 실사용 톤
- 과한 감탄 금지 ("혁신적인 AI 도구!" X → "이거 생각보다 쓸만함" O)
- 개발자 밈/유머 OK ("야근", "레거시", "it works on my machine")

## 용어 수준
- 전문 용어 설명 없이 사용 OK
- 약어 OK (PR, CI, DB, API 등)
- 코드 스니펫 짧게 포함 가능

## 이 타겟에서 잘 먹히는 스타일
- insight: 기술 팁, 새 도구 소개, GitHub 트렌딩
- curated-list: 개발 도구 족보, 라이브러리 추천
- relatable: 개발자 일상 공감 (버그, 야근, 코드리뷰)
- self-deprecation: 개발자 자조 (레거시 코드, 기술 부채)
